# End-to-End-Streaming-Data-Pipeline-on-AWS

<img width="1614" height="871" alt="End-to-End data streaming Pipeline on AWS" src="https://github.com/user-attachments/assets/774dc5ab-b22d-4997-a7e0-659ac144fa1f" />


An enterprise-style real-time data engineering pipeline built using Apache Kafka, Confluent Platform, AWS, and Tableau.

The project demonstrates how event data generated through REST APIs is validated using Schema Registry, streamed into a Kafka cluster, stored in Amazon S3 using Kafka Connect, transformed using AWS Lambda, cataloged by AWS Glue, queried with Amazon Athena, and visualized in Tableau.

---

## Tech Stack

### Streaming

- Apache Kafka (KRaft Cluster)
- Confluent Platform 7.8.0
- Kafka REST Proxy
- Kafka Connect (Distributed Mode)
- Schema Registry
- Confluent Control Center

### Cloud

- Amazon EC2
- Amazon S3
- AWS Lambda
- AWS Glue
- Amazon Athena
- Amazon SNS

### Analytics

- Tableau

### API Testing

- Postman
- REST APIs
